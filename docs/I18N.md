# Internationalization (i18n)

How to add a new language to Hum2Song Studio. Contributors can add a language by adding one JSON file and one manifest entry.

---

## Overview

Hum2Song Studio uses a plugin-friendly i18n module (`static/i18n/`). The language switch dropdown appears in the Inspector header. Switching language does **not** reload the page and preserves all state (selected clip, open editor, instrument, optimize options, etc.).

---

## Adding a new language

### 1. Add locale file

Create a new JSON file under `static/i18n/locales/` named by language code:

```
static/i18n/locales/
├── en.json
├── zh.json
└── <your-lang-code>.json
```

Use the same keys as `en.json` (the source of truth). Example:

```json
{
  "common.ok": "OK",
  "common.cancel": "Cancel",
  "common.loading": "Loading...",
  "common.error": "Error",
  "common.language": "Language"
}
```

For flat keys like `common.ok`, either format works:

- **Flat:** `{ "common.ok": "OK" }`
- **Nested:** `{ "common": { "ok": "OK" } }`

### 2. Add manifest entry

Edit `static/i18n/manifest.json` and add your language:

```json
[
  {"code":"en","label":"English"},
  {"code":"zh","label":"中文"},
  {"code":"ja","label":"日本語"}
]
```

The dropdown is populated from this manifest. The `code` must match your locale filename (e.g. `ja` → `locales/ja.json`).

## API (browser)

The module exposes `window.I18N`:

| Method | Description |
|--------|-------------|
| `I18N.getLang()` | Current language code |
| `I18N.setLang(lang)` | Set language, persist to localStorage |
| `I18N.t(key, params?)` | Translation lookup with fallback to English then key |
| `I18N.register(lang, dict)` | Register a dict in memory |
| `I18N.load(lang, opts?)` | Fetch and register locale from `/static/i18n/locales/<lang>.json` |
| `I18N.loadManifest(opts?)` | Fetch and cache manifest from `/static/i18n/manifest.json` |
| `I18N.availableLanguages()` | Manifest list or default `[{en},{zh}]` |
| `I18N.init()` | Set lang from localStorage or navigator.language |

**Storage:** `localStorage.hum2song_studio_lang` — do not change this key.

---

## Notes

- Switching language triggers a minimal UI refresh (no page reload).
- Missing keys in a locale fall back to English, then to the key itself.
- `I18N.t()` never throws.
